## 1. 전략 개요

### 1.1 투자 철학

본 전략은 다음 가설에 기반한다:

> **"한국 시장의 시가총액 상위 10개 기업은 시장 전체의 흐름을 대변하며, 장기 보유 시 KOSPI 지수를 상회하거나 근접한 수익률을 안정적으로 추구할 수 있다."**

이 가설을 구현하기 위해 다음 원칙을 적용한다:

- **단순성 우선**: 결정적 알고리즘만 사용. LLM·기술적 지표·뉴스 분석 모두 배제.
- **위험 분산**: 10개 종목 균등 평가금액 보유를 지향.
- **유동성 우선**: 시총 상위 = 거래량 풍부 = 슬리피지 최소.
- **자동화**: 평일 09:00 + 매시간 정시 무인 rebalance.

### 1.2 비교 대상 (Why Not...)

| 대안 전략 | 본 전략과의 차이 |
|---|---|
| KOSPI 200 ETF (예: KODEX 200) | 종목 수 200개 대 10개. 더 광범위하지만 운용 보수 0.15%/년 발생 |
| 단일 종목 집중 (예: 삼성전자 100%) | 폭등기에는 큰 이익, 폭락기에는 자산 절반 손실 위험 |
| 12-Rule 매매 (구 시스템) | 복잡한 룰셋(섹터·캔들·뉴스·LLM 의존), 운영 비용 큼 |
| 가치 투자 (PER/PBR 스크리닝) | 시점 판단 필요, 매수/매도 타이밍 복잡 |

본 전략은 **운영 단순성**과 **시장 노출 충분성** 사이의 균형점.

```

  ↓
[시총 정렬 → Top 10 추출]
  ↓

  ↓

  ↓

  ↓

  ↓
[평가금액 최저 종목 재분배 매수 (1주씩 반복)]
  ↓
[로그 + 시스템 이벤트 기록]
```

## 3. 매매 알고리즘 상세

### 3.1 의사 코드

```text
function runRebalance():
    top10 = fetchTop10()                    # Naver 모바일 API
    holdings = getCurrentHoldings()          # SQLite + KIS 잔고

    # 1단계: 매도 (Top 10 이탈자) — 시장 브레이크 무시
    for h in holdings:
        if h.ticker not in top10:
            sellMarket(h.ticker, h.quantity)

    # 2단계: 매수 가능 여부 확인
    brake = checkMarketBrake()               # KOSPI 지수, VIX 체크
    if brake.shouldBrake:
        log("시장 브레이크 — 매수 차단")
        return

    cash = getOrderableAmount()              # KIS 가용 현금

    # 3단계: 미보유 시총 우선 매수 (1주씩)
    for s in top10 sorted by rank ascending:
        if s.ticker in holdings: continue
        if s.price > cash:
            log("시총 #{rank} skip — 잔고 부족")
            continue
        buyMarket(s.ticker, 1)
        cash -= s.price
        holdings[s.ticker] = 1

    # 4단계: 보유 재분배 (평가금액 최저 + 매수 가능)
    while cash > 0 and iteration < 30:
        candidates = [
            s in top10
            where holdings[s.ticker] > 0 and s.price <= cash
        ]
        if candidates is empty: break

        target = candidates sorted by (qty * price) ascending [0]
        buyMarket(target.ticker, 1)
        cash -= target.price
        holdings[target.ticker] += 1
```

### 3.2 시총 Top 10 산정 방식

**소스**: Naver Finance Mobile JSON API

```
GET https://m.stock.naver.com/api/stocks/marketValue/KOSPI?pageSize=30&page=1
GET https://m.stock.naver.com/api/stocks/marketValue/KOSDAQ?pageSize=30&page=1
```

응답에서 추출:
- `itemCode` (6자리 코드)
- `stockName`
- `marketValueRaw` (원 단위 시가총액, 정렬 기준)
- `closePriceRaw` (현재가/종가)
- `fluctuationsRatio` (등락률)

**산정 절차**:
1. 두 시장에서 각각 30위까지 fetch
2. 60종목 통합 → `marketValueRaw` 내림차순 정렬
3. 상위 10개 추출 → 1~10위 부여
4. 1시간 캐시 (cron이 매시 정시 force refresh)
5. 양쪽 API 실패 시 stale 캐시 fallback

**우선주 포함**: 보통주/우선주 구분 없음. 5/12 시점 Top 10에 삼성전자우(005935) 포함됨.

### 3.3 매수 우선순위 — 시총 순위 기반

| 결정 | 이유 |
|---|---|
| 시총 순위 1위부터 시도 | 시장 대표성, 유동성 최고 |
| 1주 가격 > 잔고 시 skip (다음 순위 시도) | 결정적 — "기다리지 않고" 다음 종목 진입 |
| skip된 고가 종목은 다음 라운드에 자동 재시도 | Top 10 이탈 매도로 누적된 잔고에서 진입 가능 |

**예시 (라운드 시작 시 잔고 100만원)**:
- 1위 삼성전자 (28만) → 매수 ✓ (잔고 72만)
- 2위 SK하이닉스 (183만) → skip (잔고 < 가격)
- 3위 삼성전자우 (19만) → 매수 ✓ (잔고 53만)
- 4위 SK스퀘어 (113만) → skip
- ... 계속

### 3.4 보유 재분배 — 평가금액 균등화

미보유 매수 완료 후 잔고가 남으면:

1. 보유 중인 Top 10 종목들의 평가금액 (수량 × 현재가) 계산
2. 평가금액 최저 + 1주 가격 ≤ 잔고 인 종목 1주 추가 매수
3. 1주도 못 살 때까지 반복 (안전장치: 최대 30회)

**효과**: 시간이 지남에 따라 보유 종목 평가금액이 균등화. 단 가격 자체가 다른 종목 N배인 경우 (예: SK하이닉스 vs 두산에너빌리티)는 완전 균등 불가.

### 3.5 시장 브레이크 (안전망)

매수 단계 전에 체크:

| 지표 | 임계값 | 동작 |
|---|---:|---|
| KOSPI 일간 등락률 | ≤ -2.0% | 신규 매수 차단 |
| VIX 지수 | ≥ 30 | 신규 매수 차단 |

**중요**: 매도(Top 10 이탈)는 시장 브레이크 무시. 이탈은 항상 즉시 정리.

## 4. 백테스팅 방법론

### 4.1 데이터

**시총 산정용 후보 풀**:
- 2026-05-12 기준 KOSPI Top 50 + KOSDAQ Top 50 = 100종목
- 각 종목의 발행주식수는 `marketValueRaw / closePriceRaw` 로 역산하여 6개월간 변동 없다고 가정

**일별 종가**:
- API: `https://api.stock.naver.com/chart/domestic/item/{ticker}/day?startDateTime=...&endDateTime=...`
- JSON 응답: localDate, closePrice, openPrice, highPrice, lowPrice, accumulatedTradingVolume

### 4.2 시뮬레이션 규칙

| 항목 | 규칙 |
|---|---|
| 매주 월요일 가정 | 정상 거래일이면 그대로, 휴장이면 다음 거래일로 자동 이월 |
| 시총 산정 시점 | rebalance 일자 종가 기준 |
| 매수 가격 | rebalance 일자 종가 (슬리피지 없음 가정) |
| 매도 가격 | rebalance 일자 종가 |
| 수수료 | 시뮬레이션에서 미반영 (실제 KIS 수수료 약 0.015% + 거래세 0.20% 별도) |
| 시장 브레이크 | 시뮬레이션 단순화 — 매수 항상 시도 |

### 4.3 한계 및 가정

1. **발행주식수 6개월간 고정** — 실제로는 액면분할/유상증자 등 변동 가능 (대형주는 드물게)
2. **5/12 기준 100종목 후보 풀** — 6개월 전 Top 10이 이 풀 밖이었다면 누락 가능. KOSPI 시총 1~30위는 거의 변동 없어 영향 미미
3. **수수료/거래세 미반영** — 실제 수익률은 시뮬레이션 대비 약 -1~-2%p 차감 예상
4. **종가 매수 가정** — 실제로는 09:00 시초가에 매수 시 슬리피지 영향 있음

### 6.2 격차 발생 원인

본 기간(2025-11 ~ 2026-05)은 한국 시장에서 매우 특수한 강세장:
- 시총 1위 삼성전자: +177% (반도체 사이클 회복)
- SK하이닉스: +76% (HBM 수요)
- SK스퀘어: +94%
- LG에너지솔루션: +10% (전기차 둔화)
- 종목별 격차 매우 큼

본 전략은 "평가금액 균등"을 추구하므로 단일 종목 폭등을 따라가지 못한다. 이는 **알고리즘의 결함이 아닌 의도된 트레이드오프**:

| 시장 상황 | 단일 종목 (예: 삼성전자) | Top 10 분산 |
|---|---|---|
| 시총 1위 폭등 | **이익 극대화** | 부분 이익 |
| 시총 1위 폭락 | **자산 절반 손실 위험** | 분산 효과로 안전 |
| 시장 횡보 | 정체 | 정체 |
| 광범위 약세 | 단일 종목 더 큰 손실 | 평균 손실 |

### 6.3 적립식 vs 일시 투입

같은 기간(약 1~6개월) 비교 시:

| 투입 방식 | 수익률 | 평균 매입 단가 효과 |
|---|---:|---|
| 1개월 적립 (5회) | +18.12% | 후반부 가격에 가까움 |
| 3개월 적립 (13회) | +24.47% | 평균 매입 단가 효과 강화 |
| 6개월 일시 투입 | +55.61% | 첫날 시점 운이 좋음 (저점 매수) |

**강세장에서는 일시 투입 > 적립식**. 그러나 **하락장에서는 적립식 > 일시 투입**.

### 6.4 거래 비용 미반영

위 결과는 모두 수수료/세금 **미반영** 시뮬레이션:
- KIS 매수 수수료: 약 0.015%
- KIS 매도 수수료: 약 0.015%
- 증권거래세 (매도시): 0.20%
- 합산: 매수+매도 1사이클당 약 0.23%

3개월 시뮬레이션 거래 34건 × 평균 약 0.23% = 약 -7.8% 차감 예상 → 실제 수익률 약 +16~17% (시뮬레이션 +24.47% 대비)

6개월 시뮬레이션 거래 48건 × 0.23% = 약 -11% 차감 → 실제 약 +44~45% (시뮬레이션 +55.61% 대비)

## 7. 알고리즘 강점과 한계

### 7.1 강점

| 항목 | 설명 |
|---|---|
| **결정적 동작** | LLM·기술적 지표 의존 0. 같은 입력에 항상 같은 출력 |
| **유동성 안전** | 시총 상위 = 호가창 풍부 = 슬리피지 최소 |
| **자가 회복** | 한 종목 휴장/상장폐지 → 다음 라운드에 자동 대체 |
| **고가 종목 자연 진입** | 이탈 매도로 누적된 잔고가 SK하이닉스 같은 비싼 종목 매수에 사용됨 |
| **운영 단순** | 외부 API 2개 (KIS + Naver), 설정 9개 필드, 코드 ~3,500줄 |
| **연산 가벼움** | 모바일 워치에서도 돌아갈 수준 |
| **하락장 안전** | 분산 효과 — 단일 종목 폭락에 대한 자산 보호 |

### 7.2 한계

| 항목 | 설명 | 완화 방안 |
|---|---|---|
| **강세장 underperform** | 시총 1위 폭등 시 분산으로 인해 따라가지 못함 | 의도된 트레이드오프, 알고리즘 변경 시 위험 분산 약화 |
| **고가 종목 일시 미진입** | SK하이닉스 등이 초기에는 매수 불가 | 시간이 지나면 누적 자금으로 자연 진입 |
| **거래 빈도** | Top 10 변동 시마다 매도/매수 발생, 거래세 누적 | 수수료/세금이 실제 수익률에 -7~-11%p 영향 |
| **시총 1위 편향** | 평가금액 균등 목표가 시총 1위 비중을 낮춤 | 사용자 결정 — 강세장 추종이 목표라면 가중치 추가 가능 |
| **자동 잔고 활용 한계** | 단일 라운드에서 SK하이닉스 1주 가격(183만) 모자란 경우, 그 라운드에선 진입 불가 | 적립식이면 다음 라운드, 일시 투입이면 이탈 매도 대기 |

### 7.3 수익률 기대치 정리

| 시장 상황 | 본 전략 예상 6개월 수익률 | 비고 |
|---|---:|---|
| **극강세 (시총 1위 +100% 이상)** | +30~60% | 본 백테스트 |
| **일반 강세** | +15~30% | 평균적 강세장 |
| **횡보** | -5~+5% | 거래세로 인한 마이너스 |
| **약세 (-10~-20%)** | -10~-15% | 분산 효과로 단일 종목 대비 안전 |
| **극약세 (-30% 이상)** | -20~-25% | 시장 브레이크가 -2% 임계값에서 매수 차단 |

## 8. 결론 및 운영 가이드

### 8.1 결론

1. **알고리즘은 운영 안정성 검증 완료** — 1/3/6개월 백테스트 모두 양의 수익률
2. **위험 분산 vs 단일 종목 폭등 추종은 트레이드오프** — 본 전략은 분산 우선
3. **연산 부담 미미** — 모바일/워치에서도 실행 가능
4. **외부 API 의존 최소** — KIS + Naver 2개만으로 완전 운영

### 8.2 추천 운영 시나리오

**적합한 사용자**:
- 장기 안정 수익 추구 (연 15~30% 목표)
- 단일 종목 리서치 시간 부족
- 자동매매 시스템에 LLM 비용 부담스러움
- KOSPI 광범위 노출 원하나 ETF 운용 보수 회피

**부적합한 사용자**:
- 단기 단일 종목 트레이딩 선호
- 시장 타이밍 매매 시도
- 시총 1위 폭등을 100% 추종하고 싶은 경우

### 8.3 운영 체크리스트

배포 후 첫 평일 09:00 동작 확인:

```bash
# NAS에서
docker logs --since "10m" stock-manager 2>&1 | grep -E "Top10|Scheduler|cron"

# 기대 출력:
# [Scheduler] Top10 cron 등록 (09:00 + 10~14시 매시간 rebalance, 15:25 force-market, 15:50 reconcile)
# [Scheduler] 총 4개 cron 활성화
# [Scheduler] 09:00 Top10 rebalance — sold: X, bought: Y
```

월간 점검 항목:
- [ ] Top 10 구성원 변화 추적 (들어오고/나간 종목)
- [ ] 평가금액 분포 균형 (특정 종목 N배 편중 여부)
- [ ] 시장 브레이크 발동 빈도 (KOSPI -2% 빈도)
- [ ] KIS API 에러 발생 여부 (system_events)
- [ ] 거래 수수료/세금 누적 (예상 대비 차이)

### 8.4 향후 개선 후보 (선택 사항)

| 우선순위 | 항목 | 효과 |
|---|---|---|
| 낮음 | DB 미사용 테이블 DROP (watch_targets 등) | 디스크 절약, 마이그레이션 위험 |
| 낮음 | 모바일 PWA화 | 모바일 모니터링 편의성 |
| 중간 | 거래 수수료 시뮬레이션 반영 | 실제 수익률 추정 정확도 |
| 중간 | 시장 브레이크 임계값 튜닝 | 백테스트로 최적값 도출 |
| 검토 필요 | 시총 1위 가중치 옵션 | 알고리즘 변경 — 의도된 트레이드오프 변화 |
